## 常见问题与解决方案

### BM25 检索相关性问题

**问题**：查询"现金流分析"只返回现金流量表内容，缺少综合损益表等关联报表。

**原因**：
1. BM25 是关键词匹配，不具备财务分析知识
2. LLM 扩展的关键词太抽象（如"經營活動之現金流入"），文档中不存在
3. 分词不一致：查询词"營業收入"被分词为一个词，但文档"香港車務營運收入"被分词为另一个整体

**解决方案**：
1. 使用 LLM 扩展查询，自动关联相关财务报表
2. LLM prompt 需要提供文档实际词汇示例，避免生成抽象概念
3. 财务词典需要包含细粒度词汇，确保分词一致性

**经验教训**：
- LLM 扩展的关键词必须是文档中**实际存在的词汇**
- 提供领域特定词汇示例能显著提高扩展质量
- jieba 分词结果依赖词典，需要针对领域定制

### jieba 缓存问题

**问题**：修改财务词典后，分词结果没有变化。

**原因**：jieba 会缓存分词模型到 `/tmp/jieba.cache`，修改词典后需要清除缓存。

**解决方案**：
```bash
rm -f /tmp/jieba.cache
```

## 最佳实践

### LLM 查询扩展设计

1. **提供标准财务科目示例**：在 prompt 中给出财务报告通用术语，避免 LLM 生成抽象概念
2. **使用繁体中文**：财务报告（尤其是香港/台湾公司）通常使用繁体中文
3. **简洁输出**：只要求返回关键词列表，不要解释

### 财务词典维护

1. **简体编写，加载时转繁体**：便于维护，自动适配繁体文档
2. **包含标准科目词汇**：营业收入、营业成本、净利润、总资产等通用术语
3. **定期更新**：根据检索效果补充新词汇
