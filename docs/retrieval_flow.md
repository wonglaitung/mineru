## 三大安全阀

| 安全阀 | 位置 | 说明 |
|--------|------|------|
| **A: 硬截断** | `max_loops` 参数 | 最多迭代 3 次，防止死循环 |
| **B: 物理去重** | `merged = {}` 按 line 去重 | 防止多轮迭代导致 Token 膨胀 |
| **C: 格式降级** | `_parse_eval_response()` | LLM 输出格式崩溃时正则捕获，终极兜底强制终止 |

## 各步骤详解

### Step 1: 表格识别与切分

**目的**：确保表格完整性，避免关键数据被切断。

**实现**：
- 使用 `md_parser.py` 解析 Markdown AST
- 识别 `<table>`、`|---|` 等表格标记
- 表格整体作为一个 chunk
- 非表格区域按段落分块（默认 500 字符）

### Step 2: Parent-Child 标签生成

**目的**：为大表格生成简短检索标签。

**实现**：
- 纵向提取表格第一列的核心科目
- 组合成 Child 标签：`"表格标题_科目1/科目2/科目3"`
- Parent 保留完整表格内容

**示例**：

| Parent（完整表格） | Child（检索标签） |
|-------------------|------------------|
| 經營活動現金流量,7438,8979<br>經營活動產生的現金,29,7438<br>... | 綜合現金流量表_經營活動現金流量/經營活動產生的現金 |

### Step 3: 权重增强型混合索引

**目的**：提升 Child 标签的 BM25 分数，同时保留 Parent 内容。

**公式**：
```
enriched_text = (child_label + " ") × 3 + parent_text
```

**效果**：
- Child 标签重复 3 次 → TF 权重提升
- Parent 内容保留 → 科目检索不漏失

### Step 4: LLM 查询扩展

**目的**：将用户查询扩展为相关关键词列表。

**实现**：
- 调用 `qwen_engine.py` 的 `chat_with_llm()`
- 自动关联三大财务报表
- 返回繁体中文关键词

**示例**：
```
输入: "现金流分析"
输出: ["現金流量表", "經營活動", "投資活動", "籌資活動", "營業收入", "淨利潤"]
```

### Step 5: 多关键词 BM25 检索

**目的**：每个关键词独立检索，扩大召回范围。

**实现**：
- 使用 `rank_bm25.BM25Okapi`
- jieba 中文分词 + 财务词典增强
- 每个关键词检索 Top-10 块

### Step 7: LLM 质检闭环（多轮迭代）

**目的**：确保不遗漏任何关键上下文，自动穿透嵌套附注。

**实现**：
- 拼接当前片段池，发送给质检 LLM
- LLM 判断资料是否齐备，返回 `COMPLETE` 或 `INCOMPLETE`
- 如果不齐备，LLM 返回 `next_query` 进行下一轮检索
- 物理去重，新块加入片段池

**质检提示词要点**：
- 角色：财报资料核对员
- 任务：检查原文是否齐备，不分析问题
- 输出：JSON 格式 `{status, reason, next_query}`

**示例**：
```
第 1 轮质检:
❌ 资料不齐: 缺失"附注五 投资收益明细表"
next_query: "附注五 投资收益"

第 2 轮检索:
BM25("附注五 投资收益") → 新增 3 个块

第 2 轮质检:
✅ 资料已齐备: 涉及的报表及附注已全部集齐
```

### Step 8: 表格分数权重调整

**目的**：补偿表格 BM25 分数偏低的问题。

**原因**：
- 表格标题只出现一次 → TF 低
- 表格内容长 → IDF 分母大

**实现**：
```python
if chunk['metadata'].get('has_table'):
    chunk['score'] *= TABLE_SCORE_MULTIPLIER  # 默认 2.0
```

### Step 9: Token 预算控制与重排

**目的**：控制输出 Token 数量，适配 LLM 上下文限制。

**实现**：
- 按行号去重，保留最高分数
- 按分数排序，累加 Token 直到上限
- 按原始顺序重排，便于阅读

### Step 10: 输出上下文

**格式**：
```
[来源: 行 XXX, 章节: XXX]
内容...

---

[来源: 行 XXX, 章节: XXX]
内容...
```
