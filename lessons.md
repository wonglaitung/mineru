## 常见问题与解决方案

### BM25 检索相关性问题

**问题**：查询"现金流分析"只返回现金流量表内容，缺少综合损益表等关联报表。

**原因**：
1. BM25 是关键词匹配，不具备财务分析知识
2. LLM 扩展的关键词太抽象（如"經營活動之現金流入"），文档中不存在
3. 分词不一致：查询词"營業收入"被分词为一个词，但文档"營運收入"被分词为另一个整体

**解决方案**：
1. 使用 LLM 扩展查询，自动关联相关财务报表
2. LLM prompt 需要提供文档实际词汇示例，避免生成抽象概念
3. 财务词典需要包含细粒度词汇，确保分词一致性

**经验教训**：
- LLM 扩展的关键词必须是文档中**实际存在的词汇**
- 提供领域特定词汇示例能显著提高扩展质量
- jieba 分词结果依赖词典，需要针对领域定制

### BM25 关键词匹配问题

**问题**：LLM 扩展的关键词"總資產"、"總負債"在文档中 BM25 分数为 0。

**原因**：
- 文档中只有"資產"、"負債"等单独词汇，没有"總資產"这样的组合词
- BM25 是精确匹配，"總資產"不会匹配"資產"

**解决方案**：
1. LLM prompt 要求生成简短关键词，避免组合词
2. 示例中明确说明：如"總資產"改为"資產"

**经验教训**：
- 关键词要尽量简短，避免组合词
- 测试关键词是否存在于文档中，调整 prompt

### 表格块重叠污染问题

**问题**：表格块内容被前一个块的末尾内容污染，如綜合損益表块开头包含无关内容。

**原因**：重叠添加逻辑会把前一个块的末尾内容添加到当前块的开头。

**解决方案**：跳过表格块，不添加重叠内容。

**经验教训**：
- 表格块应该保持干净，不添加重叠内容
- 修改重叠逻辑时，需要考虑特殊块类型

### 三大财务报表检索遗漏问题

**问题**：BM25 检索时，綜合損益表、財務狀況表等关键表格没有被选中。

**原因**：
1. BM25 分数基于词频，表格标题只出现一次，分数低
2. Token 预算控制时，低分数的块被排除

**解决方案**：
1. 为表格块增加 BM25 分数权重（翻倍）
2. 这是通用方案，适用于所有表格块

**经验教训**：
- 表格标题通常只出现一次，需要调整 BM25 分数权重
- 避免硬编码特定报表名称，使用通用规则

### jieba 缓存问题

**问题**：修改财务词典后，分词结果没有变化。

**原因**：jieba 会缓存分词模型到 `/tmp/jieba.cache`，修改词典后需要清除缓存。

**解决方案**：
```bash
rm -f /tmp/jieba.cache
```

### CSV 表格切分问题

**问题**：大型财务报表表格被切分成多个小块，导致检索不完整。

**原因**：
1. CSV 表格检测条件 `len(line.strip()) > 10` 过于严格，短行（如"下列人士應佔:,,,"）被判定为非表格行
2. 逗号数量差异容忍度 `> 2` 过小，财务报表常有空单元格导致逗号数量变化

**解决方案**：
1. 放宽 CSV 行判定条件：短行但有多于 3 个逗号也认为是表格
2. 放宽容忍度：逗号数量差异超过 5 才认为表格结束

**经验教训**：
- 财务报表表格常有空单元格和短行，需要更宽松的检测条件
- 测试不同表格结构，调整参数

### 表格标题识别问题

**问题**：现金流量表等表格块没有包含标题，导致 BM25 分数偏低。

**原因**：某些财务报表标题没有 `#` 前缀（PDF 转换问题），表格切分逻辑只查找以 `#` 开头的标题。

**解决方案**：扩展标题识别逻辑，同时查找"独立短行"（长度 5-30 字符、逗号少、前后有空行）。

**经验教训**：
- PDF 转换可能丢失标题格式，需要备用识别机制
- 使用通用启发式规则，避免硬编码特定名称

## 架构设计

### 通用检索器 vs 领域检索器

**设计原则**：分离通用逻辑和领域特定逻辑。

**实现**：
- `bm25_retriever.py` - 通用 BM25 检索器，适用于任何 Markdown 文档
- `financial_retriever.py` - 财务分析检索器，继承 BM25Retriever，添加 LLM 查询扩展

**配置参数**（类常量，可被子类覆盖）：
```python
TABLE_TITLE_SEARCH_RANGE = 200      # 表格前搜索标题的字符范围
TABLE_TITLE_MIN_LEN = 5             # 标题最小长度
TABLE_TITLE_MAX_LEN = 30            # 标题最大长度
TABLE_TITLE_MAX_COMMA = 1           # 标题最大逗号数
TABLE_TITLE_MAX_DISTANCE = 50       # 标题距离表格的最大距离
TABLE_CONTEXT_AFTER = 100           # 表格后上下文长度
PARAGRAPH_SEARCH_RANGE = 100        # 段落边界搜索范围
```

## 最佳实践

### LLM 查询扩展设计

1. **提供标准财务科目示例**：在 prompt 中给出财务报告通用术语，避免 LLM 生成抽象概念
2. **使用繁体中文**：财务报告（尤其是香港/台湾公司）通常使用繁体中文
3. **简洁输出**：只要求返回关键词列表，不要解释
4. **强制包含三大报表**：在代码中确保"綜合損益表"、"財務狀況表"、"現金流量表"始终在关键词列表中
5. **关键词要简短**：避免组合词（如"總資產"改为"資產"）

### 财务词典维护

1. **简体编写，加载时转繁体**：便于维护，自动适配繁体文档
2. **包含标准科目词汇**：营业收入、营业成本、净利润、总资产等通用术语
3. **定期更新**：根据检索效果补充新词汇
4. **添加完整报表名称**：如"综合损益表"、"财务状况表"、"现金流量表"作为完整词汇加入词典

### 表格切分设计

1. **表格块不添加重叠**：保持表格内容干净
2. **包含表格前的标题**：从最近的标题行开始，不包含过多无关内容
3. **限制上下文长度**：避免包含过长的无关内容
